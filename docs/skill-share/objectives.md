# System Design: Automated, Selective AI Skill Distribution Network (`agentskills.io` Standard)

## 1. Core Architecture Overview

The system maps a single, private monolithic GitHub repository (`company-ai-skills`) directly to the local personal skill directories scanned by AI desktop clients (**OpenCode, Codex, and Claude Desktop/Cowork**).

Instead of copying files manually, each machine runs a headless local Git clone that silently synchronizes with the remote cloud directory in the background.

* **Remote Source of Truth:** `github.com/your-org/company-ai-skills`
* **Target Local Injection Path (macOS):** `~/.claude/skills/` (Also universally mapped via `~/.agents/skills/` or symlinked based on tool configuration).
* **Target Local Injection Path (Windows):** `%USERPROFILE%\.claude\skills\`

---

## 2. Selective Sync Mechanism (The Opt-In/Opt-Out Manifest)

To manage a large library (e.g., 60+ skills) without overwhelming individual users or bloating Claude’s local context window, the system implements **Git Sparse-Checkout (Cone Mode)**.

Instead of downloading all 60 folders, Git uses a local "manifest" to determine which directories are allowed to materialize on the user's hard drive. The rest remain safely in the cloud.

### How it Works Behind the Scenes

* Git maintains a local manifest file at `.git/info/sparse-checkout`.
* When a background sync occurs, Git queries this file. It updates and pulls *only* the directories explicitly listed in the manifest.
* If a skill folder is omitted from a user's manifest, it will not exist on their machine, and their local Claude instance will never see or activate it.

### The UI Wrapper for Non-Technical Users

Because the 5 team members are non-technical, they should never edit Git configs. The script will expose a simple configuration wrapper.
To change what skills they use, they run a one-line command or a clickable terminal shortcut that updates their profile:

```bash
# Example command executed via an automation shortcut to set their active skill list
git sparse-checkout set core-utilities operations-invoice-parser delivery-pdf-generator

```

## 3. The Background Sync Engine

A lightweight, silent system daemon (a LaunchAgent on macOS or a Task Scheduler task on Windows) executes every **10 minutes** to keep the active skills updated without human intervention.

### The Automated Synchronization Sequence

The daemon runs the following automated loop in the background:

```bash
#!/bin/bash
cd ~/.claude/skills/ || exit

# 1. Stash any local manual modifications to references/ or local assets to prevent blocks
git stash --include-untracked

# 2. Fetch changes from remote repository
git fetch origin main

# 3. Pull updates down using rebase to maintain a clean linear timeline
# Note: Because sparse-checkout is enabled, this only pulls files for matched manifest folders
git pull --rebase origin main

# 4. Pop their local tweaks back on top seamlessly
git stash pop

```

## 4. Master `.gitignore` Configuration

To prevent local chat logs, caching, or auto-generated user assets from polluting the main repository, the root of the GitHub repository must enforce a strict `.gitignore`.

```text
# Ignore OS-specific metadata junk
.DS_Store
Thumbs.db

# Ignore any local dynamic runtimes or caches generated by execution steps
__pycache__/
*.pyc
.venv/
node_modules/

# Ignore custom user log outputs if they run files locally
*.log
local-debug/

```

## 5. Critical System Constraints & "Gotchas"

When handing this blueprint to an implementation agent, ensure it builds safeguards around these exact scenarios:

### Gotcha 1: Silent Authentication Failures

* **The Risk:** If a user’s network drops, or a GitHub credential token expires, the background script will stall out waiting for a password entry, locking up the daemon process.
* **The Mitigation:** The deployment script must configure authentication using long-lived **SSH Deploy Keys (read-only)** tied to the specific machine, or a persistent personal access token stored securely in the system keychain. The sync script must include an initial internet check step (`nc -zw1 google.com 443`) and immediately terminate silently if a connection isn't present.

### Gotcha 2: Merge Conflicts on Reference Files

* **The Risk:** A user’s Claude Cowork agent updates a shared template tracking sheet inside a skill's `references/` folder at the same time you push an update to that same file on GitHub. `git pull --rebase` will fail with a conflict.
* **The Mitigation:** The background script must evaluate the exit code of the pull command. If `exit_code != 0`, the script must **abort the merge immediately** (`git rebase --abort`), pause the synchronization sequence, and fire a native system notification to the user (e.g., using `osascript` on macOS) stating: *"AI Skill Update Paused: Local file conflict detected. Please contact your administrator."*

### Gotcha 3: Claude Client Directory Caching

* **The Risk:** Git successfully updates a `SKILL.md` file in the background, but an active, running instance of Claude Desktop or Cowork keeps the old prompt instructions cached in RAM.
* **The Mitigation:** Most modern agents dynamically reload on file modification changes. However, if caching occurs, the sync script can touch the root directory configuration file or look for a native reload command to force the AI app to re-index its available skills.

## 6. Full Setup Execution Script (Blueprint for the Agent)

This is the boilerplate installation sequence the builder agent should package into an installation tool for your team:

```bash
# 1. Target the correct system directory
mkdir -p ~/.claude/skills
cd ~/.claude/skills

# 2. Initialize a clean local repository and link it to the organization source
git init
git remote add origin git@github.com:your-org/company-ai-skills.git

# 3. Initialize Sparse Checkout in modern Cone Mode
git sparse-checkout init --cone

# 4. Define the baseline skills that EVERY user gets by default
git sparse-checkout set core-utilities onboarding-assistant

# 5. Execute the initial shallow pull to minimize space
git fetch --depth=1 origin main
git checkout main

```
