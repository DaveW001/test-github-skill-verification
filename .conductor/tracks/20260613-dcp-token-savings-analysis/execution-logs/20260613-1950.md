## Understanding the Multiplier Effect

DCP (Dynamic Context Pruning) compresses conversation context, reducing the tokens sent to the model. The **one-time savings** (2.09M tokens, 1.07%) represents the direct reduction from compression.

The **compound savings** (64.6M tokens, 33.15%) accounts for the cascading effect: when DCP compresses earlier messages, all subsequent requests in the session benefit from the smaller context window. With an average of ~40 requests per session that has DCP, the small one-time savings compounds into significant total reduction.

**Multiplier = 30.9x** means each token saved by DCP effectively saves 30.9 tokens across the session's lifetime.
