```powershell

$text = Get-Content -Raw -LiteralPath $path
```

The reviewer-applied fix uses a unique marker string (`"    conductor-track-executor: allow"`) that appears **exactly once** in the file (verified). Body prose is preserved verbatim.

### Evidence 2 - Tasks 2.1 / 2.2 silently no-op

Hex of real executor file's first 12 bytes: `2D 2D 2D 0A 6E 61 6D 65 3A 20 63 6F` = `---\nname: co...` (LF only).

Command under review:
```powershell
$text = $text.Replace("model: zai-coding-plan/glm-5.2", "model: zai-coding-plan/glm-5.1")
if (-not $text.Contains("hidden: true")) {
    $text = $text.Replace("---`r`n", "---`r`nhidden: true`r`n")
}
$text = $text.Replace("You are a Conductor track execution specialist.", "You are a Conductor track execution specialist.`r`n`r`nFallback tier: ...")
```

Dry-run result on a temp copy of the real file:
- After `.Replace("---`r`n", ...)` : `hidden: true` **not present** in `$text` (the file is LF, not CRLF).
- After `.Replace("You are a Conductor track execution specialist.", ...)` : the source phrase is **not found**; the real body is `You are the **Conductor Track Executor** (Stage 4). Use the track's \`plan.md\`...`. The replace is a silent no-op.

Net effect: the produced `conductor-track-executor-glm51.md` would have `model: zai-coding-plan/glm-5.2` (unchanged), no `hidden: true`, and no Tier note. Acceptance would fail (the model line is wrong).

The reviewer-applied fix uses `---\nname:` as a unique opening-fence marker (appears once; the closing fence is followed by body content, not by `name:`) and appends the Tier note at the end of the file using `[string]::Replace()` on a stable trailing anchor.
