## Changes introduced by Stage 2 that need correction before execution

1. **0.0 does not verify its own stated condition.** It accepts Node 22.5+ by version but does not prove that `node:sqlite` loads with the required flag. The diagnostic probe, not the version parse, should be authoritative.
2. **0.1 `-WhatIf` isolation is incorrect.** The check tests `candidate-manifest.json` at a path unrelated to `$outDir`; a stale real artifact can produce a false failure, while output in the declared directory can escape detection. Use a newly created empty temporary directory and assert it remains empty.
3. **Comment-only false assurance.** `Select-String -SimpleMatch` does not establish that required terms occur in executable code; comments can satisfy the check. Behavioral fixture tests must prove read-only open, field redaction, family closure, stable ordering, and manifest binding.
4. **Authorization remained implicit.** Stage 2 tightened manifest approval but did not clearly constrain current execution to inventory-only. This was the highest-confidence safety fix and has been applied.
5. **The first-task pointer became stale.** Stage 2 inserted task 0.0 but left “First Task to Execute” at 0.1. This has been corrected.
