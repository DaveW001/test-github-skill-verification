# Workflow

## Development Approach
- Dogfood `.conductor/` structure while building.
- Keep specs and plans in repo as source of truth.

## Session Management
- Use full tracks for features or non-trivial fixes.
- Use quick notes in the active track for small questions.

## Plan Tracking
- `plan.md` is authoritative.
- Update checkbox states: [ ] pending, [~] in progress, [x] complete.

## Commit Style
- Conventional Commits (default), unless updated later.

## Validation
- Run targeted tests for modified areas.
- Confirm `plan.md` tasks are complete before closing a track.

## Definition of Done
- All acceptance criteria met.
- `plan.md` tasks are checked off.
- `tracks.md` entry updated with summary and status.
