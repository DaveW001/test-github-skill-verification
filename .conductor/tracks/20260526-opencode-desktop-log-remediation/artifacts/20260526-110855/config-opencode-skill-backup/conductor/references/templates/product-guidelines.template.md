# Product Guidelines

## Voice and Tone
- Direct, helpful, and concise.
- Output should explain next steps when errors occur.

## UX Principles
- Favor clarity over cleverness.
- Provide actionable output for each command.

## CLI Output Patterns
- Start with a short outcome line.
- Follow with 1-3 bullet points when needed.

## Accessibility
- Avoid relying on color alone to convey status.
